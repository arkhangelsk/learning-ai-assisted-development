**The 70% Problem with AI Coding Tools**

AI can get you about 70% of the way to a working solution very quickly.

The code compiles. It runs. It even looks reasonable.

But the remaining 30%—making it production-ready—is where most of the real engineering work happens.

That’s where you deal with edge cases, security issues, performance, and how everything fits into your system. It’s also where you decide whether the solution actually solves the right problem.

In practice, the workflow looks like this:
- AI generates a solid first draft  
- You review and find gaps  
- Iteration adds complexity  
- The final stretch requires careful, manual work  

One pattern that stands out: experienced engineers tend to get more value from these tools. They can evaluate output quickly, spot issues, and guide the tool effectively.

Less experienced developers are more likely to accept code that looks correct but has hidden problems.

AI helps with execution. It doesn’t replace judgment.

The practical approach:
- Use AI for the first 70%  
- Focus your effort on the final 30%  
- Treat review and iteration as essential, not optional  

The tools are useful, but the responsibility for quality still sits with the engineer.
